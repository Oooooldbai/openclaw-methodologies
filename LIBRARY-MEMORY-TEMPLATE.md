# 图书馆记忆法 - 核心思想

> 建立索引 → 按需加载 → 减少token消耗

---

## 核心思想

图书馆记忆法的核心思想是：**像图书馆一样管理记忆**。

### 类比：图书馆

- **图书馆不会**：把所有书都放在阅读桌上（token消耗大）
- **图书馆会**：建立索引系统 → 按需取书 → 读完放回

### 类比：AI记忆

- **不要**：把所有记忆文件都加载到上下文（token消耗大）
- **要**：建立索引系统 → 按需加载记忆 → 加载完释放

---

## 解决的问题

### 问题1：启动慢
- 每次启动都加载所有memory文件
- Token消耗大，启动慢

**解决方案**：只加载轻量索引（<500 tokens）

### 问题2：信息丢失
- 启动时忘记项目配置
- 启动时忘记关键信息

**解决方案**：关键配置永久存储在MEMORY.md

### 问题3：无法按需加载
- 需要某个知识时，不知道在哪个文件
- 需要逐个搜索memory文件

**解决方案**：建立关键词映射表

---

## 核心原则

### 1. 索引优先
- 启动时只加载索引文件
- 不加载所有记忆文件

### 2. 按需加载
- 根据用户提及的关键词
- 动态加载对应的专题文件

### 3. 轻量存储
- 索引文件：<500 tokens
- 专题文件：按需加载

### 4. 永久存储
- 关键配置存储在MEMORY.md
- 启动时优先加载MEMORY.md

---

## Token节省效果

| 场景 | 传统方式 | 图书馆记忆法 | 节省 |
|------|---------|--------------|------|
| 癿书馆记忆法**：解决"启动问题"
- **消防队模式**：解决"执行问题"

**结合使用**：
- 图书馆记忆法存储项目配置
- 消防队模式存储任务进度
- 两者结合，形成完整的记忆+任务管理体系
