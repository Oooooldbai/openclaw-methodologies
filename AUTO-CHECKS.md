# 自动检查规则

> 不依赖"记得"，依赖HEARTBEAT触发

---

## 每次HEARTBEAT必执行

### 1. 检查点完整性
```
遍历 checkpoints/memory/
每个文件的 synced_to 必须全为 true
有 false → 立即修复 → 标 true
```

### 2. 日志生命周期
```
检查 memory/ 下超过7天的日志
有 → 蒸馏（有价值→topics/）+ 归档（→archive/）

检查 memory/archive/ 下超过30天的日志
有 → 删除
```

### 3. 仓库同步状态
```
检查所有本地仓库未push的commit
有 → push（最多重试2次，失败记录日志）
```

### 4. HEARTBEAT有效性
```
检查HEARTBEAT中的任务是否过期
过期 → 标记 ✅ 或删除
新增任务 → 及时写入
```

---

## 每周自检（周日）

- [ ] MEMORY.md 是否超过2KB？
- [ ] 方法论模板是否膨胀？（消防队<200行，图书馆<150行）
- [ ] 配置是否泄露敏感信息？
- [ ] 检查点自动扫描是否正常运行？

---

## 为什么必须自动化

- AI每次会话重启，记忆清零
- 日志、检查点、配置会遗忘
- **唯一的办法：让HEARTBEAT强制触发，不依赖自觉性**

## 熔断机制

连续3次HEARTBEAT发现**同一问题**未修复 → 停止自动修复，发消息通知用户。

适用场景：检查点synced_to反复false、仓库push反复失败等。

**为什么需要：防止Agent在循环修复中浪费Token，让人类及时介入。**
